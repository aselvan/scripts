# Contributing to scripts

Thank you for your interest in contributing. Please follow these basic guidelines.

## How to Contribute
1. Fork the repository.
2. Create a new branch from master.
3. Make your changes and ensure they are well-tested.
4. Submit a pull request with a clear explanation of your work.

## Standards
* Ensure code is clean and adheres to existing formatting.
* Provide comments for complex logic or shell script optimizations.
* Update any relevant documentation if your changes introduce new functionality.

## Issues
If you find a bug or have a feature request, please open an issue on the GitHub issue tracker before starting work.
